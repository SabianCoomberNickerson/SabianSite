Copyright, Creative Common (CC) Licenses, Fair Use

Copyright

Copyright is a tool to recognise work as your own. It is applied automatically and lasts for your lifetime +50 years.

CC Licenses:

  1. Attribution (BY)
  2. Non-Commercial (SA)
  3. No Derivatives (NC)
  4. Share Alike (ND)

NZ Law

New Zealand does not have "fair use" it has "fair dealing". This gives a third party the ability to use your work without your permission if it is for research, private study, criticism, review or news reporting.

Privacy & Web Privacy Policy

Privacy Policy

A privacy policy is a legal document an organisation has that tells people what data they gather, why it is gathered, how it is used, who it might be shared with and what security measures are in place to keep it secure. An organisation that collects user's data is legally required to have a privacy policy.

The NZ privacy act 2020 has 13 principles that govern how an organisation can collect and use data

  1. Purpose for collection
  2. Source of information
  3. What to tell the individual about
  4. Manner of collection
  5. Storage and Security of information
  6. Providing people access to their information
  7. Correction of personal information
  8. Ensure accuracy before using information
  9. Limits on retention of personal information
  10. Use of personal information
  11. Disclosing personal information
  12. Disclosure outside New Zealand
  13. Unique identifiers

Web Security

Common Cyber Attacks

  1. Malware

    2. Harmful software like viruses can steal data or disrupt services.

    Use anti-malware software and keep it updated

  2. Phishing

    3. Deceptive messages such as emails used to trick users into giving up sensitive information

    Employees need to be taught to be mindful of suspicious links, and accounts need to have strong passwords.

  3. SQL injection

    4. Malicious SQL code that allows attackers to modify databases.

    Queries need to be parameterised so user inputs are treated as data and not code.

  4. Cross-site scripting

    5. Malicious scripts that are injected into websites that target legitimate users when their browsers read the site.

    Sanitise and validate all user data before displaying it on the site.

    Use secure coding practices, such as output encoding, to prevent scripts from being executed.

    Implement a Content Security Policy to define trusted sources for content.

  5. DDoS

    6. Overwhelming a server with traffic from multiple sources preventing legitamite users from accessing the site.

    Use reputable DDoS protection services, implement a firewall to filter malicious traffic and ensure there is sufficient network infrastructure to handle high traffic.
