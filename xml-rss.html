XML Syntax Rules

6 important rules that must be adhered to in order to create a valid XML file:

Rule 1: Documents must have a root element

There must be a root element that is the parent of all other elements.


<root>
    <child>
        <subchild>...</subchild>
    </child>
</root>

Rule 2: All elements must have a closing tag

(The XML prolog does not have a closing tag. It is not part of the XML document.)

In this example there is an opening and closing p tag:

<p>This is a paragraph<p>

Rule 3: Elements must be properly nested

An element opened inside another element, must be closed in that same element

The following is an example of what NOT to do:

<b><i>This text is bold and italic!<b><i>

This is what you should do (see how the italics tag has to be closed before closing the bold tag because it was opened inside the bold tag):

<b><i>This text is bold and italic!<i><b>

Rule 4: Attribute values must always be quoted

Some elements can have attributes, these must be put in quotes.

<element attributeName="attributeValue">...</element>

Rule 5: XML is case sensitive

This means capital letters are important. The word "potato" is not the same as "Potato" for example.

Rule 6: Prolog

The XML prolog is optional, but if it's used it must be at the top of the document.

<?xml version="1.0" encoding="UTF-8"?>
